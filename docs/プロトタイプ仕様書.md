# dance, unleashed プロトタイプ仕様書

## 1. 概要
- ジャンル: ワンボタン・リズムアクション
- プレイ人数: 1人
- 対応環境: デスクトップブラウザ優先、モバイルブラウザ対応は簡易
- 1ラン時間: 144秒
- 勝利条件: ノーミスで 144 秒を完走する
- 失敗条件: キック判定を 1 回でもミスする
- 設計方針:
  - ゲーム要素はキックを刻み続けることに限定する
  - 解放人数は演出値として扱い、複雑な得点計算は行わない
  - ミス時は進行を継続せず、新しい144秒ランを最初から開始する

## 2. コアループ
1. 4拍のカウントイン後、楽曲とビート進行を開始する。
2. プレイヤーはキック入力で毎ビートに合わせて入力する。
3. 成功すると連続成功数が伸び、固定の解放演出カーブに沿って群衆と音が膨らむ。
4. 必要なら補助パーカッションを自由に鳴らしてグルーヴを足せる。
5. ミスするとそのランは終了し、現在解放人数、連続成功数、追加トラックをリセットする。
6. 短い失敗演出の後、新しい144秒ランを最初から再開始する。
7. ノーミスで 144 秒終了時にクリア表示を出す。

## 3. 入力仕様
### キック入力
- 判定対象:
  - キーボード `Space`
  - キーボード `Enter`
  - メインパッドのクリック
  - メインパッドのタップ
- 1ビート中に受け付ける成功入力は1回のみ。
- 成功入力後の追加入力は無効化する。
- 判定窓の外でキック入力した場合はミスとして扱う。

### 補助パーカッション入力
- 例:
  - キーボード `A`: スネア
  - キーボード `S`: ハイハット
  - キーボード `D`: クラップ
  - 画面上の補助パッドタップ
- 補助パーカッション入力はいつ押してもよい。
- 補助パーカッション入力は成功失敗に影響しない。
- 補助パーカッション入力は音と軽い視覚反応だけを返す。

## 4. リズム仕様
- BPM: 125 固定
- 拍子: 4/4
- 総アクティブビート数: 300
- 開始手順:
  - タイトル画面で入力を受けたら AudioContext を開始する
  - 4拍の視覚カウントインを表示する
  - 1拍目からゲーム本編開始
- ミス後の再開始手順:
  - 失敗演出を短く表示する
  - 残り時間と進行ビートを初期化する
  - 2拍から4拍程度の短いカウントを入れて新しいランを開始する

## 5. 判定仕様
- 判定対象: 各ビートの基準時刻
- 判定幅:
  - `Kick`: 基準時刻から ±90ms
  - `Miss`: 上記以外
- 各ビートで一度も成功入力がないまま `+90ms` を超えた場合、自動で `Miss`
- UI表示上は `On Beat` / `Early` / `Late` / `Miss` を出し分ける
- 補助パーカッション入力には判定を出さない

## 6. 状態管理
### プレイ中に保持する値
- `timeRemaining`: 残り時間
- `beatIndex`: 現在ビート番号
- `kickStreak`: 連続成功数
- `currentCrowd`: 現在解放人数
- `phase`: 演出フェーズ
- `lastJudgement`: 直近判定

### ミス時のリセット対象
- `timeRemaining`
- `beatIndex`
- `kickStreak`
- `currentCrowd`
- `phase`
- 追加BGMトラックの有効状態

### ミス時に維持してよい値
- `attemptCount`
- `sessionStartTime`

## 7. 解放演出カーブ
- 解放人数は報酬計算ではなく、連続成功数に応じて進む固定の演出値とする。
- 実装では `kickStreak` をもとにフェーズと目標人数を決め、フレームごとに補間表示する。
- 1ラン中にのみ有効であり、ミス時は 0 からやり直す。

| 連続成功数 | 表示人数の目安 | 意図 |
| --- | ---: | --- |
| 1 - 16 | 0 - 500 | 静かな立ち上がり |
| 17 - 48 | 500 - 5,000 | 解放が見え始める |
| 49 - 96 | 5,000 - 50,000 | 中盤への移行 |
| 97 - 160 | 50,000 - 250,000 | 視覚密度が上がる |
| 161 - 220 | 250,000 - 700,000 | 終盤の熱量形成 |
| 221 - 300 | 700,000 - 1,000,000 | フィナーレへ到達 |

- 各範囲の内部では線形補間またはイージングで滑らかに増やす。
- 数字の厳密性より「増え方の説得力」を優先する。
- バランス調整時は人数そのものより、フェーズ遷移の気持ちよさを優先する。

## 8. 演出フェーズ
- フェーズは `kickStreak` で決定する。

| フェーズ | 連続成功条件 | 視覚変化 | 音変化 |
| --- | --- | --- | --- |
| 0 | 0 | 暗い背景、少人数 | キックのみ |
| 1 | 16+ | 観客の増加、ライト点灯 | ベース追加 |
| 2 | 48+ | 背景色変化、粒子追加 | ハイハットループ追加 |
| 3 | 96+ | カメラ揺れ、フラッシュ増加 | コード追加 |
| 4 | 160+ | 群衆密度上昇、空間の広がり | シンセ追加 |
| 5 | 220+ | 最大照明、全画面の熱量演出 | リード追加 |

- ミス時はフェーズ 0 に即時戻す。
- フェーズ遷移時に短い成功演出を入れる。
- 補助パーカッションはどのフェーズでも鳴らせるが、ミックス上は主役のキックを邪魔しない。

## 9. 画面仕様
### タイトル画面
- タイトル表示
- `Press Space / Tap to Start`
- ルールの短文表示
  - `Keep the beat for 144 seconds`
  - `One miss resets the crowd`

### プレイ画面
- 中央: ビートリングまたはパルス表示
- 背景: 群衆密度と照明演出
- 上部左: 現在解放人数
- 上部中央: 残り時間
- 上部右: 連続成功数
- 下部中央: 直近判定テキスト
- 下部または左右端: 補助パーカッションパッド
- 画面隅: 現在の挑戦回数を小さく表示してもよい

### 失敗演出
- `MISSED`
- 群衆と追加トラックを即時消去
- 直後に短い再開カウントを表示

### クリア表示
- `UNLEASHED`
- 144秒完走
- `Retry`

## 10. 群衆表現仕様
- 実人数を全てスプライト化しない。
- 表現方法:
  - 少人数時: 個別のシルエット点
  - 中人数時: グリッドまたは扇状に集約表示
  - 大人数時: 密度レイヤー、パーティクル、ライトの面表現
- 画面上の表示数は端末負荷に応じて 100 から 1,500 要素程度に制御する。
- `currentCrowd` は見た目の密度、揺れ、発光量に変換して表示する。

## 11. 音響仕様
- Web Audio API を利用する。
- 構成:
  - 常時: キック
  - フェーズ1: ベース
  - フェーズ2: ハイハットループ
  - フェーズ3: コード
  - フェーズ4: シンセパッド
  - フェーズ5: リード
- 補助パーカッション:
  - 任意入力でスネア、ハイハット、クラップを鳴らせる
  - 発音は短く、同時発音数は制限する
- ミス時:
  - 追加トラックを即座に停止または短いフェードで消す
  - 新しいラン開始時にキックから再構築する
- プロトタイプ段階では短いループ素材または簡易シンセでよい

## 12. 技術仕様
- 描画: `CanvasRenderingContext2D`
- 更新: `requestAnimationFrame`
- 時間管理:
  - オーディオ時刻を基準にビートを計算する
  - 見た目は描画フレームで補間する
- 構成案:
  - `Game`: 状態遷移とゲームループ
  - `BeatScheduler`: ビート時刻管理
  - `JudgeSystem`: キック入力判定
  - `AudioManager`: 音声制御
  - `PercussionPad`: 補助パーカッション入力
  - `Renderer`: HUDと背景演出描画
- パラメータは1か所の設定オブジェクトに集約する。

## 13. 成功指標
- 初見でルールを理解できる。
- ミス時に「144秒やり直し」の重さが明確に伝わる。
- 成功が続くと「止まりたくない」心理が生まれる。
- 終盤の人数増加と音圧の上昇が明確に感じられる。
- 補助パーカッションが楽しいが、なくてもゲームとして成立する。
- 少なくともデスクトップChrome系ブラウザで安定動作する。
